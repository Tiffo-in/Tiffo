## 2024-11-20 - [Database Query Optimization in Partner Earnings Calculation]
**Learning:** In the `getEarnings` function inside `backend/src/controllers/partnerController.js`, multiple independent queries (`Payment.find`) were used to calculate earnings metrics for different timeframes (today, yesterday, this week, last week, this month, last month, all time). Fetching full document lists sequentially for each interval is highly inefficient, leads to memory overhead, and generates excess database load.
**Action:** Replace multiple `find()` queries with a single MongoDB `$aggregate` pipeline using conditional summing (`$sum` with `$cond`). This delegates calculation directly to the database, massively reducing data transferred over the wire and solving a significant N+1-like querying anti-pattern. Next time I notice repetitive analytical queries fetching arrays only to sum a field in memory, I'll leverage MongoDB's aggregation pipeline.
